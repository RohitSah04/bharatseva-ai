# /backend/models

SQLAlchemy ORM model classes. One file per table (or one file for closely related tables, e.g., auth tokens).

## Model Map

| File | Tables |
|---|---|
| `user.py` | `users`, `refresh_tokens`, `password_reset_tokens` |
| `citizen_profile.py` | `citizen_profiles` |
| `scheme.py` | `schemes` |
| `saved_scheme.py` | `saved_schemes` |
| `eligibility_check.py` | `eligibility_checks` |

| `agent_log.py` | `agent_logs` |

| `kb_source.py` | `kb_sources` |

## Conventions

- All primary keys are UUID v4 strings generated by the application (not DB auto-increment).
- All `created_at` / `updated_at` fields use `datetime.utcnow` as `default` / `onupdate`.
- Audit tables (`eligibility_checks`, `agent_logs`) must have **no** `update` or `delete` operations in the ORM layer — enforce via service-layer convention and documented in each model's docstring.
